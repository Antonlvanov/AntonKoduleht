MVP (Minimaalselt Elujõuline Toode)

MVP on toote minimaalne versioon, mis võimaldab testida kasutajate huvi. IT projektides kasutan MVP põhimõtet järgmiselt:

  • Põhifunktsionaalsuse kiire väljastamine
  • Kasutajate tagasiside kogumine
  • Ressursside optimeerimine
  • Täiendavate funktsioonide lisamine järk-järgult

Kuidas luua kaugrepo

Kaugreposid võimaldavad meil hoida oma koodi turvaliselt ja jagada seda teistega. Siin on samm-sammult juhised:

1. Loo uus repo GitHubis

  • Logi sisse GitHubi kontosse
  • Klõpsa nuppu "New"
  • Anna reposidile nimi ja kirjeldus
  • Vali "Public" või "Private"
  • Ära lisa README.md faili veel

2. Initsialiseeri kohalik repo

git init

3. Lisa failid

git add .

4. Tee esimene commit

git commit -m "Esialgne versioon"

5. Lisa kaugrepo

git remote add origin https://github.com/kasutajanimi/reponimi.git

6. Saada muudatused

git push -u origin main

NB! Kui kasutad vanemat Git versiooni, võib peamine haru olla nimega "master" mitte "main".
